## 🚀 如何使用

### 1. 主工作流（外贸客户开发主工作流）

**功能**：
- ✅ 自动搜索客户
- ✅ 获取邮箱
- ✅ 发送邮件
- ❌ **不发送运行结果通知**

**触发方式**：

#### 自动运行
- 每天 UTC 1:00（北京时间 9:00）自动运行

#### 手动触发
1. 访问：https://github.com/aapeakinc-hue/miga-auto-workflow/actions
2. 点击 **"外贸客户开发主工作流"**
3. 点击 **"Run workflow"**
4. 选择分支：`main`
5. 点击绿色的 **"Run workflow"** 按钮

**查看结果**：
- 在 GitHub Actions 页面查看运行日志
- 下载上传的日志 artifact

---

### 2. 通知测试工作流（测试通知功能）

**功能**：
- ✅ 测试 resend 模块安装
- ✅ 测试 requests 模块安装
- ✅ 发送测试通知邮件

**触发方式**：
1. 访问：https://github.com/aapeakinc-hue/miga-auto-workflow/actions
2. 点击 **"测试通知功能"**
3. 点击 **"Run workflow"**
4. 点击绿色的 **"Run workflow"** 按钮

**查看结果**：
- 查看每个步骤的详细输出
- 检查是否收到测试邮件

## 🔍 常见问题

### Q: 为什么删除了通知功能？
A: 通知模块在 GitHub Actions 环境中安装失败，导致整个 workflow 失败。为了先保证核心功能正常运行，暂时移除了通知功能。

### Q: 如何知道工作流是否成功？
A:
1. 访问 GitHub Actions 页面
2. 查看最新的运行记录
3. 检查是否所有步骤都成功（绿色勾）
4. 下载并查看日志文件

### Q: 如何恢复通知功能？
A:
1. 先修复 "测试通知功能" workflow
2. 确保模块可以正常安装
3. 将通知步骤添加到主工作流中
